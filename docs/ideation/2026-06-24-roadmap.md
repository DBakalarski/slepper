## Poza fazami — „Co nowego" po deployu 🟡

Spec: `docs/superpowers/specs/2026-06-24-changelog-update-banner-design.md`.

- [x] **Banner „co nowego" + wymuszony restart** — `public/changelog.json` (kurowany per deploy), `features/changelog/changelog.ts` (+test), `useChangelogUpdate.ts` (open+focus, no-store, +static-invariants), `components/ChangelogBanner.tsx` (+test), montaż w `app/(app)/_layout.tsx`, `vercel.json` (wyjątek rewrite + no-cache). Scala 2+ niewidziane deploye.
- [ ] **Utrzymanie:** przy każdym deployu z user-facing zmianą dopisać wpis do `changelog.json` (inkrement `v` + nadać `version` semver wg wagi zmiany: minor dla ficzera, patch dla poprawki) **oraz** zsynchronizować `version` w `app.json` i `package.json` z najnowszym wpisem changelogu. Wyświetlana wersja w Ustawieniach czyta `app.json` (`Constants.expoConfig.version`); historia zmian czyta `changelog.json`. Spec: `docs/superpowers/specs/2026-06-24-changelog-history-versioning-design.md`.
  - ⚠️ **Pułapka cache (2026-06-25):** bez wpisu w changelogu banner „co nowego" NIE pokazuje deployu (`selectUnseen` bierze tylko `v > lastSeen`) — deploy musi być ogłoszony, inaczej user zostaje na starym bundlu. Dodatkowo `version` z `app.json` jest wpiekane do bundla webowego (`APP_MANIFEST`) i **Metro cache'uje to w `$TMPDIR/metro-cache`** — sam bump `app.json` + `expo export` daje bundle ze STARĄ wersją. Po zmianie wersji build z `--clear` (`expo export --platform web --clear`) albo skasuj `$TMPDIR/metro-cache`. Vercel buduje na zimnym cache → produkcja OK, ale lokalna walidacja wersji wymaga `--clear`.
